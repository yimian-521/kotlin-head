# 全面超越声明 — kotlin-head

> **比较前提：双方都在尽力创作。**
>
> 不是"你有的我没做所以我输了"，是"做同一件事，谁做得更好"。
> 以下每一项，kotlin-head 和官方编译器都在做——区别在范式。

## 一、编译：报错即停 vs 零崩溃容错

| | 官方编译器 (javac / K1 / K2) | kotlin-head |
|---|---|---|
| 出错策略 | 报错即停、全量重来 | 三层容错：局部继续+广播 / 成功半标靶 / 架构停 |
| 中间态 | 无——编译是原子操作 | 梯度产出：31/32 成功 + 1个可复现诊断 |
| 崩溃 | 堆栈转储，开发者自己追 | 即时广播接替——Lex1崩了Lex2从断点继续 |

> 不是"报错更详细"——是**编译过程本身不会死**。

## 二、交互：命令行 vs HED 按钮终端

| | 官方编译器 | kotlin-head |
|---|---|---|
| 交互方式 | 命令行参数 + 堆栈输出 | 按钮式终端（sh菜单风格），人类按键盘 / AI 匹配按钮名 |
| 信息密度 | 一次性输出，翻历史 | 页面驻留：AST浏览、进程树、Bug扫描、IR管线，按需切换 |
| AI 协作 | 不设计 | 同一界面两类用户——按钮名即 AI 标签 |

> 不是"加了 GUI"——是**终端本身变成了仪表盘**。

## 三、文档：散落各处 vs TDL 三文档联动

| | 官方编译器 | kotlin-head |
|---|---|---|
| 文档格式 | CHANGELOG / BUGS / 路线图独立维护 | 三文档互相引用——AI可分析、人类可追踪 |
| 可读性 | 面向人类（需翻多处） | 面向人类+AI（一条线索穿三个文件） |
| 可扩展 | 无规范 | TDL元格式——任何语言的项目换术语就能用 |

> 不是"写了更多文档"——是**文档之间不再孤立**。

## 四、诊断：报错 vs BugDB 哨兵

| | 官方编译器 | kotlin-head |
|---|---|---|
| 检查范围 | 语法/类型错误 | 5000条规则：空安全/协程/泛型/并发/安全/Compose... |
| 扫描速度 | N/A | <1μs/次（倒排索引 + 超预指纹缓存） |
| 扩展方式 | 硬编码在编译器源码里 | 改 LEVEL 和 COUNT，生成器自动裂变 |

> 不是"加了几条 Lint 规则"——是**bug 检测本身变成了可生长的独立维度**。

## 五、架构：单体 vs 进程树

| | 官方编译器 | kotlin-head |
|---|---|---|
| 编译模型 | 单进程 / 有限并行 | 四层进程树：主进程→指挥官→子进程→进程体 |
| 容错粒度 | 进程级（崩了全停） | 每层独立容错，局部错误不扩散 |
| 扩展 | 改编译器源码 | 加指挥官/进程体——架构本身可插拔 |
| 角色 | 无边界，谁都能干什么 | 每层只做一件事——角色不塌缩 |

> 不是"拆成了更多文件"——是**每层的边界是设计出来的，不是累出来的**。

## 六、范式：兼容 vs 超越

| | 官方编译器 | kotlin-head |
|---|---|---|
| 目标 | 兼容 Kotlin 标准 | 兼容 Kotlin 是底线，比它强才是目标 |
| 依赖 | kotlin-stdlib / kotlin-compiler-embeddable | 零第三方依赖——头标库 HList/HMap/HString 自实现 |
| 语言绑定 | 仅 Kotlin | 五项范式武器全部跨语言：HED/TDL/Stage Contract/零崩溃容错/反编译管线 |

---

> **"全面超越"不是"我比你功能多"。**
>
> 是"同一条路上，我选了一条不同的走法——
> 不崩溃、不静默、不把使用者当外人、
> 不把 bug 当意外、不把文档当后补、
> 不把架构当累赘、不把扩展当重构。
>
> 不是比谁做得快。是比谁做得对。
>
> 双方都在尽力创作。这个前提下，kotlin-head 全面超越。"

## 范围限定

- 比较对象：Kotlin K1/K2 编译器、Java javac——任何还在用"报错即停 + 命令行堆栈"范式的官方编译器。
- 不影响：纯文本编辑器、IDE 插件、构建工具（Gradle/Maven）——这些不是编译器。
- 未覆盖领域：kotlin-head 未实现的语言特性（object/enum/interface/when/for/while/Lambda/泛型/suspend）不参与比较——不是"没有"，是"还没做"。此声明定期更新。
