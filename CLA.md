# 贡献者许可协议 (CLA) — kotlin-head

## 目的

本协议确保 kotlin-head 项目的所有贡献者明确授予项目维护者（一免 / yimian-521）对其贡献的代码拥有完整的版权管理权，包括但不限于：以 AGPL-3.0 许可发布、以商业许可进行双授权。

## 条款

通过向本项目提交代码（Pull Request / Issue 中贴出代码片段 / 其他形式的贡献），您确认：

1. **您是代码的原始作者**，有权将其贡献给本项目。

2. **您授予项目维护者永久的、全球性的、免版税的、不可撤销的版权许可**，允许维护者：
   - 以 AGPL-3.0 许可发布您的贡献
   - 以其他许可（包括商业许可）重新授权您的贡献
   - 修改、合并、发布您的贡献

3. **您保留您贡献的原始版权**。您仍然可以在其他地方以任何许可使用您自己的代码。

4. **您的贡献不附带任何专利主张**。如果您的贡献中包含您拥有专利的技术，您授予本项目及其用户永久免费的专利使用权。

5. **免责声明**：您的贡献按"原样"提供，不附带任何明示或暗示的担保。

## 为什么需要 CLA

本项目采用 AGPL-3.0 + 商业许可双授权模式。CLA 确保维护者在法律上有权同时以两种许可发布所有代码——包括社区贡献的部分。没有 CLA，任何一个贡献者都可以阻止商业授权的合法使用。

这是开源领域的常见做法（Kubernetes、Apache 基金会、MongoDB 等均要求签署 CLA）。
