# TSRC 提交完整模板

## TSRC 官网五个独立栏位

TSRC 表单有五个独立输入框，每个单独填写：

| 序号 | 栏位 | 说明 |
|------|------|------|
| 1 | 漏洞名称 | `[蓝鲸CI] 文件名 漏洞简述(CWE编号)` 不超过50字符 |
| 2 | 漏洞类型 | Web漏洞→大类→子类 |
| 3 | 请求数据包 | HTTP格式 |
| 4 | 响应数据包 | HTTP格式 |
| 5 | 漏洞描述 | 1-2-3-4完整格式（会重复URL等，必须再写一遍） |

## 栏位1: 漏洞名称

```
蓝鲸CI 文件名 漏洞简述(CWE编号)
```

限制：50字符以内

## 栏位2: 漏洞类型速查

| 漏洞实质 | Web大类 | 子类 |
|----------|---------|------|
| Token写日志/URL | Web漏洞 | 敏感信息明文储存 |
| API无权限检查 | Web漏洞 | 权限绕过→越权 |
| 熔断fallback跳过认证 | Web漏洞 | 权限绕过→越权 |
| 交接API无身份验证 | Web漏洞 | 权限绕过→越权 |
| 用户枚举/接口暴露 | Web漏洞 | 内部敏感接口暴露 |
| 拦截器user==null放行 | Web漏洞 | 权限绕过→越权 |
| 注册端点无权限 | Web漏洞 | 权限绕过→越权 |

## 栏位3: 请求数据包

```
POST /api/端点路径 HTTP/1.1
Host: example.com
# 源码审计漏洞，非实际HTTP请求
# 审计文件：文件名.kt 第X-Y行
# 无@BkApiPermission注解
```

## 栏位4: 响应数据包

```
HTTP/1.1 200 OK
# 漏洞存在于代码层面
# 触发后效果：一句话描述
```

## 栏位5: 漏洞描述（1-2-3-4格式）

```
============================================================
1. 漏洞描述
漏洞文件：完整文件名.kt
漏洞URL：https://github.com/TencentBlueKing/bk-ci/blob/master/完整路径
路径：端点路径
关键参数：触发参数（无则写"无"）
权限注解：无@BkApiPermission / @PreAuthorize
需要测试账号：否
需要工具：无（源码审计）

2. 复现步骤
第一步：构造请求（描述）
第二步：请求到达端点
第三步：无权限检查直接处理
第四步：返回200 OK，操作完成
源码审计：定位代码 → 确认无权限注解 → 描述触发条件

3. 危害说明
攻击者可以：
- 具体危害1
- 具体危害2

4. 修复建议
- 具体修复1
- 具体修复2

============================================================
发现方法：免免安全审计法
GitHub Issue：TencentBlueKing/bk-ci #编号
联系方式：微信 il123445 | 手机 13040850045
============================================================
```

## 合并提交（同类漏洞打包）

同类漏洞可合并为一个提交：
- 名称写系列名：`伪装模块系列:插件+ITSM授权劫持-无权限检查(CWE-306)`
- URL填其中一个主要文件
- 描述中两个都写全
