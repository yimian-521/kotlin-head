##TSRC 表单字段对应实际漏洞类型（Web漏洞子类）

| 漏洞实质 | 选 TSRC 类型 |
|----------|-------------|
| Token写日志/URL | 敏感信息明文储存 |
| API无权限检查 | 权限绕过 |
| 熔断fallback跳过认证 | 未鉴权 |
| 交接API无身份验证 | 权限绕过 |
| 用户枚举/接口暴露 | 内部敏感接口暴露 |

##请求/响应数据包格式

源码审计必须遵守的标准格式：

请求数据包：
```
POST /api/端点路径 HTTP/1.1
Host: example.com
# 源码审计漏洞，非实际HTTP请求
# 审计文件：文件名.kt 第X-Y行
# 关键描述：无@BkApiPermission注解等
```

响应数据包：
```
HTTP/1.1 200 OK
# 漏洞存在于代码层面
# 触发后效果：一句话描述
```
